# AI Chatbot 2.0

A web-based AI chatbot using Langchain and Ollama.

## Research Sources

Development leveraged several resources:

*   **AI Assistance:** Gemini 2.5 Pro provided significant coding assistance, debugging help, and implementation suggestions throughout the development process.
*   **Web Development Tutorials:** General concepts and specific techniques were referenced from various YouTube tutorials covering Flask, JavaScript, CSS, and Fetch API usage.
*   **Ollama Documentation:** [Ollama Official Website](https://ollama.com/) for setup and model management.
*   **Langchain Documentation:** [Langchain Python Documentation](https://python.langchain.com/) for understanding LLM interaction, prompts, and streaming.
*   **PokéAPI Documentation:** [PokéAPI Official Website](https://pokeapi.co/) for understanding endpoints, data structures, and request methods.
*   **Web Standards:** MDN Web Docs for HTML, CSS, and JavaScript references.

## Core Features

*   **Streaming Chat:** Real-time conversation interface.
*   **Personas:** Choose from various chatbot personalities (Default, Pirate, Steve, etc.) or define your own.
*   **API Integration:** Ask the "Professor Oak" persona about Pokémon to get detailed entries (including forms, stats, flavor text, evolutions) fetched live from PokéAPI. Evolution questions are also answered using API data.
*   **Markdown & Code:** Bot responses support Markdown formatting and highlighted code blocks with a copy button.
*   **Reactions:** React to bot messages with emojis, and the bot will react to your emoji!.
*   **Customization:** Light/dark theme toggle and settings menu for persona/ruleset management.
*   **Easter Egg:** Type `/pong` to play a game of Pong!

## Technology Stack

*   **Backend:** Python with Flask framework (lightweight microframework for developing web apps in Python).
    *   Uses Langchain to interact with the Ollama LLM.
    *   Handles chat logic, persona switching, and PokéAPI requests.
    *   Streams responses using Server-Sent Events (SSE).
*   **Frontend:** Vanilla JavaScript, HTML, and CSS.
    *   Handles user interface, settings, theme switching, and displaying streamed responses.
    *   Uses Marked.js for Markdown rendering and Highlight.js for code highlighting.
*   **AI Model:** Ollama (running `llama3.2:3b` locally).
*   **External API:** PokéAPI for Pokémon data.
*   **Fonts:** Google Fonts (Geist Mono, Tiny5).


## Development Challenges & Solutions

Several issues were encountered and resolved during development:

*   **Server-Sent Event (SSE) Parsing:** Initially, concatenated SSE messages caused JSON parsing errors on the frontend. This was resolved by implementing a more robust buffering mechanism in JavaScript (`sseBuffer`) to process messages based on the `\n\n` delimiter.
*   **PokéAPI Form Handling:** The initial direct `/pokemon/{name}` lookup failed for Pokémon with multiple forms (e.g., Giratina). The solution involved switching to a two-step process: querying `/pokemon-species/{name}` first to get all variety URLs, then fetching data for each specific form.
*   **Evolution Data:** Displaying evolution required fetching data from the `/evolution-chain/` endpoint identified via the species data. Logic was added to parse this nested structure both for text-based evolution explanations and for showing the next evolution in the Pokédex card.
*   **CSS Specificity & Styling:** Applying specific styles (like Pokémon type background colors or persona-themed reaction bubbles) sometimes required increasing CSS specificity or ensuring correct class names were being generated by JavaScript (e.g., `.toLowerCase()` for type tags).
*   **Font Inheritance:** Input fields and buttons didn't automatically inherit the default `body` font (`Geist Mono`). Explicitly setting `font-family: inherit;` on these elements in CSS fixed this.
*   **JavaScript Scope Errors:** Variables related to the Pong game (`isPongRunning`, `resizePongCanvas`) were initially used before being declared in the correct scope, leading to `ReferenceError`s. Declaring them globally resolved these.
*   **Layout Adjustments:** Fine-tuning the layout of elements like message reactions and Pokédex card sections required adjustments to Flexbox/Grid properties and padding/margins.

## Setup & Installation

1.  **Clone Repository:** (If applicable) `git clone <repository-url>`
2.  **Python Environment:** Create and activate a virtual environment:
    ```bash
    python -m venv venv
    # Windows
    .\venv\Scripts\activate
    # macOS/Linux
    source venv/bin/activate
    ```
3.  **Install Dependencies:**
    *   Ensure you have a `requirements.txt` file with at least:
        ```txt
        Flask>=2.0
        langchain>=0.1.0,<0.2.0 
        langchain-community>=0.0.30 
        requests>=2.0
        ```
    *   Install:
        ```bash
        pip install -r requirements.txt
        ```
4.  **Install Ollama:** Download and install Ollama from [ollama.com](https://ollama.com/).
5.  **Pull Model:** Run `ollama pull llama3.2:3b` in your terminal.

## Running the Application

1.  **Start Ollama:** Ensure the Ollama application/service is running.
2.  **Run Flask App:**
    ```bash
    python app.py
    ```
3.  **Open Browser:** Navigate to `http://127.0.0.1:5000/`.
